Overview

This class represents a list of items, be it a user’s starred list, inbox, or a traditional “playlist”.

offlineStatus

Returns the offline status of the playlist. Possible values:

@property (nonatomic, readonly) sp_playlist_offline_status offlineStatus

Discussion

SP_PLAYLIST_OFFLINE_STATUS_NO
Playlist is not offline enabled.

SP_PLAYLIST_OFFLINE_STATUS_YES
Playlist is synchronized to local storage.

SP_PLAYLIST_OFFLINE_STATUS_DOWNLOADING
This playlist is currently downloading. Only one playlist can be in this state any given time.

SP_PLAYLIST_OFFLINE_STATUS_WAITING
Playlist is queued for download.

Parameters

playlistURL

The playlist URL to create an SPPlaylist for.

aSession

The SPSession the playlist should exist in.

block

The block to be called with the created SPPlaylist object.

Discussion

This convenience method creates an SPPlaylist object if one doesn’t exist, or returns a cached SPPlaylist if one already exists for the given URL.

Warning: If you pass in an invalid playlist URL (i.e., any URL not starting spotify:user:XXXX:playlist:, this method will return nil.

moveItemsAtIndexes:toIndex:callback:

Move item(s) to another location in the list.

- (void)moveItemsAtIndexes:(NSIndexSet *)indexes toIndex:(NSUInteger)newLocation callback:(SPErrorableOperationCallback)block

Parameters

indexes

The indexes of the items to move.

newLocation

The index the items should be moved to.

block

The SPErrorableOperationCallback block to be called with an NSError if the operation failed or nil if the operation succeeded.

Discussion

All indexes are given relative to the state of the item order before the move is executed. Therefore, you don’t need to adjust the destination index to take into account items that will be moved from above it.

Warning: This operation can fail, for example if you give invalid indexes. Please make sure you check the result of this method.
